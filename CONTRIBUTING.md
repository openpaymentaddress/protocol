# Contributing to OPAP

Thank you for helping improve an open payment-address protocol.

## Before opening a pull request

1. Open an issue first for a normative or compatibility-affecting change.
2. Keep protocol, schema, conformance, and reference implementation changes aligned.
3. Do not add application, wallet, provider, asset, custody, or hosting policy to the protocol layer.
4. Run `npm ci` and `npm run check`.

## Normative changes

A normative change must state its motivation, compatibility impact, specification edit, schema impact, and conformance evidence. Changes that alter record interpretation or execution-plan semantics need valid and invalid fixtures.
